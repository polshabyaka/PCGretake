# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Personal working style

The user's personal preferences live in `CLAUDE.local.md` (not checked in). Read that first. Key rule: **explain the plan in 3-6 short points and wait for approval before changing code.** Prefer the simplest beginner-friendly Unity C# solution.

## Project

- Unity **2022.3.56f1**, 2D project using the **Universal Render Pipeline** (URP 14.0.11).
- Also uses TextMeshPro, Timeline, Visual Scripting, the Unity Test Framework (package installed, no tests written yet).
- The project name `PCGretake` suggests procedural content generation work, but no gameplay code has been written yet — all script folders are placeholders.

## How to run

1. Open the project folder in **Unity Hub** with the matching editor version (2022.3.56f1).
2. Open the scene `Assets/Scenes/Main.unity` (this is the working scene — `SampleScene.unity` is the default Unity template and can be ignored).
3. Press Play in the editor.

There is no build script, no CI, and no command-line workflow. Tests, builds, and runs happen inside the Unity Editor.

## Layout

- `Assets/Scripts/` — gameplay code, split by responsibility:
  - `Core/` — shared/base code
  - `Generation/` — procedural generation
  - `Grid/` — grid/tile logic
  - `Player/` — player behavior
  - `UI/` — UI scripts
  All of these are currently empty. When adding a script, place it in the folder that matches its one responsibility (see `CLAUDE.local.md` → "one script = one clear responsibility").
- `Assets/Prefabs/` — `CellView`, `HomeView`, `PlayerView` (simple SpriteRenderer prefabs, wired to materials in `Assets/Materials/`).
- `Assets/Scenes/Main.unity` — the scene to work in.
- `Assets/Settings/` — URP render pipeline + 2D renderer assets. Don't edit by hand.
- `Packages/manifest.json` — Unity package list.
- `Assembly-CSharp.csproj` and `PCGretake.sln` are auto-generated by Unity. Don't hand-edit them.

## When adding assets

Search `Assets/Sprites/`, `Assets/Materials/`, and `Assets/Prefabs/` by name first. If multiple files match, ask the user which one to use rather than guessing.
